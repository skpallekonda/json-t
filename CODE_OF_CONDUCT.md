# Code of Conduct

## Our Commitment

JSON-T is an open, welcoming project. Everyone who participates — through issues, pull requests, discussions, or code review — is expected to treat others with respect and professionalism.

## Expected Behaviour

- Communicate clearly and constructively.
- Give and receive feedback in good faith.
- Focus disagreements on technical merit, not personal judgement.
- Acknowledge and credit others' contributions.

## Unacceptable Behaviour

Disrespectful, abusive, or exclusionary conduct of any kind is not acceptable and will result in removal from the project.

## Reporting

If you experience or witness unacceptable behaviour, please report it by opening a private issue or emailing the maintainer directly (address in `pom.xml` under `<developers>`). All reports are handled confidentially.

## Scope

This Code of Conduct applies to all project spaces — GitHub Issues, Pull Requests, Discussions, and any other venue associated with JSON-T.

## Enforcement

The project maintainer has the right and responsibility to remove, edit, or reject contributions and to ban participants whose behaviour violates this Code of Conduct.

This project follows the spirit of the [Contributor Covenant](https://www.contributor-covenant.org).
