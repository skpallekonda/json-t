Thoughts on encryption/decryption

1. Want to have 2 options for KEK determination - a. Using receiver's public key b. Using ECDH (sender keys + receiver public key), generate shared key, using the shared key as KEK
2. Want to use version number based encryption algorithm determination, that is, v1 -> AES ; v2 -> Chacha ; v3 -> ascom (something like this, recently developed for limited resources device); etc.,
3. EncryptedPayload shall have a structure
    [
        version: u16 [1 reserved bit (signed - unused) ; 4 bits for format version; 4 bits for receiver certificate version; 4 bits for encryption algo version; 2 bits unused; 1 bit for kek selection as listed in poinnt 1],,
        len_enc_dek: u32,
        len_digest: u32,
        len_iv: u16,
        len_sha: u32,
        enc_dek: byte[],
        iv: byte[],
        sha256: byte[],
        enc_content: byte[]
    ]

    Push all these into ByteBuf, and then use Base64 encode to arrive at encrypted text
4.  So, we expect clients to supply appropriate keys (public / public+private to enable decrypt/encrypt for the library)
